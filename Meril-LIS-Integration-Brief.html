2 Lab Workflow

YOUR HMIS ── Creates Investigation Order
       │
       ▼
  [API]  LIS fetches / receives order   ◄──  Patient CR Number / UHID
       │
       ▼
  LIS ── Internal Worklist generated
         (grouped by lab · sample type · priority)
       │
       ▼
  LIS ── Barcode / Sample ID generated internally
         └── Mapped to Your Order Reference Number
       │
       ▼
  [PUSH → YOUR HMIS]  Status: SAMPLE_COLLECTED
       │
       ▼
  Analyzer processes sample  (Serial / TCP / ASTM / HL7)
       │
       ▼
  [PUSH → YOUR HMIS]  Structured Results  (parameter-wise)
       │
       ▼
  Lab Doctor verifies & authorizes
       │
       ▼
  [PUSH → YOUR HMIS]  Status: AUTHORIZED
       │
       ▼
  LIS ── PDF Report generated  (branded, H/L flagged)
         ├── [PUSH → YOUR HMIS]  PDF as Base64
         ├── WhatsApp ──► Patient  (PDF attachment)
         └── Patient Portal ──► Self-download

3 APIs We Need From You

Your HMIS must expose the following endpoints for us to call:

#API EndpointPurpose
1GET /investigations?patient_id=XXXFetch pending lab orders for a patient
2POST /status-updateReceive workflow status at each stage (we call this)
3POST /resultsReceive structured parameter-wise test results
4POST /report-uploadReceive final PDF report as Base64
5GET /master/testsYour test code master list (for mapping)
6GET /master/parametersYour parameter codes + parent codes + UOM codes

4 Data We Need From You

4.1 — Setup / Credentials

FieldDescription
Base URLYour HMIS API endpoint
Auth MethodAPI Key / OAuth2 / Basic Auth
Patient KeyCR Number / UHID / ABHA ID
Hospital CodeYour facility mapping code for our branch

4.2 — Per Investigation Order

FieldDescription
Patient identifierCR Number / UHID
Patient demographicsName, DOB, gender, mobile
Order reference numberWe map our barcode to this
Test code + nameWhat test was ordered
Sample typeBlood / Urine / etc.
Referring doctorDoctor name

4.3 — For Result Mapping (Parameter Upload)

FieldDescription
Parameter CodesYour internal code per test parameter (e.g., HB, WBC)
Parent Parameter CodesGrouping code (e.g., CBC panel → individual parameters)
UOM CodesUnit of measure per parameter (g/dL, x10³/µL, U/L, etc.)
Reference RangesNormal ranges per parameter — or we use our own
⚠️  Critical: Parameter codes, parent parameter codes, and UOM codes are required before structured result upload (API #3) can be activated. Without this mapping, only status updates and PDF delivery are possible.

5 Status Codes We Send to Your HMIS

SAMPLE_COLLECTED

RESULT_SAVED
Results entered by technician
VERIFIED
Technician verified results
AUTHORIZED
Doctor signed off report
PRINTED
PDF generated & uploaded
SAMPLE_REJECTED

CANCELED
